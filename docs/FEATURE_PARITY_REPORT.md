# Feature Parity Report

## 2026-07-14 Phase 2-4 Implementation Update

This update adds Windows-native implementations for the requested Phase 2-4
areas without copying macOS shell entry points or replacing the existing
working Windows pipeline.

Implemented call paths:

- VTT: `WindowsEditPipeline.export()` -> `subtitles.vtt.write_vtt()`
- ASS: `WindowsEditPipeline.export()` -> `subtitles.ass.write_ass()`
- Emphasis ASS: `WindowsEditPipeline.export()` -> `write_ass(..., emphasize=True)`
- Subtitle finishing: `subtitles.srt.polish_cues()` splits long cues and fills
  small subtitle gaps before SRT/VTT/ASS writing.
- Transcript export: `exports.transcript` writes Markdown, text, and CSV.
- Edit diff: `exports.edit_diff` writes JSON and Markdown based on deleted
  ranges and transcript words.
- Acoustic filler review candidates: `analysis.acoustic.acoustic_filler_candidates()`
  runs from `analyze_cuts()` when STT audio is available.
- End-syllable/word protection: `timeline.protection.protected_candidate_delete_ranges()`
  protects speech boundaries for silence-derived automatic deletions.
- Breath detection/reduction: `audio.breath.detect_breath_ranges()` records
  ranges and `ffmpeg_tools.make_clean_wav()` attenuates those ranges in the
  generated clean WAV only.
- Noise floor: `audio.features.measure_noise_floor()` measures STT WAV frames
  and stores data in candidates/manifest/breath range JSON.
- Noise removal, de-essing, high-pass, compressor, LUFS, natural/podcast:
  `audio.presets.build_audio_filter_chain()` feeds FFmpeg clean WAV filters.
- Video analysis: `bibl_windows.cli analyze-video` writes loudness, silence
  ratio, orientation, and recommendation JSON.
- Automatic preset recommendation: `bibl_windows.cli recommend-preset`.
- Shorts and vertical video XML: `bibl_windows.cli shorts`.
- Optional shorts MP4 render: `shorts --render-mp4` uses FFmpeg/NVENC.
- 2-camera sync: `bibl_windows.cli sync-2cam` uses FFmpeg PCM extraction and
  numpy FFT correlation.
- Multicam XML: `bibl_windows.cli multicam-xml` creates multi-track FCP7 XML
  from keep ranges plus explicit camera offsets.

Updated classification after this implementation:

| Feature group | Updated Windows status | Notes |
|---|---|---|
| VTT / ASS / emphasis subtitles | implemented | Generated by default during `run`; can be disabled with `--no-extra-exports`. |
| Transcript export / edit diff | implemented | Markdown/text/CSV transcript plus JSON/Markdown edit diff. |
| Acoustic filler / breath / noise / deesser / audio presets | partial implementation | Conservative review/clean-output implementation; source media is untouched. |
| End-syllable protection | implemented for silence cuts | Repetition cuts remain allowed because they intentionally remove speech. |
| Video analysis / preset recommendation | implemented | Uses ffprobe, ffmpeg loudnorm, and silencedetect heuristics. |
| Shorts / vertical video | implemented | XML-first; MP4 render optional. |
| 2-camera sync / multicam XML | implemented | Sync reports offset; multicam XML uses explicit offsets and keep ranges. |

## 예상 위험 요소

- 원본 `.claude` 문서가 Windows 코드가 아닌 `engine/*.py`, `python3`, macOS 흐름을 지시한다. 사용자/agent가 그대로 따르면 실패한다.
- Windows `Path.resolve()`는 존재하지 않는 외부 경로나 네트워크 경로에서 예외/의도치 않은 절대화 문제가 생길 수 있다.
- FCP7 XML의 Windows `file:///C:/...` 경로는 공백/한글 인코딩이 맞아야 Premiere import가 안정적이다.
- `TransformersWhisperBackend`의 `return_timestamps="word"`와 `chunk_length_s=30`은 모델/transformers 버전에 따라 경고 또는 동작 차이가 생길 수 있다.
- 1시간 이상 영상 STT는 메모리/시간/VRAM 사용이 크다. 스트리밍/청크 처리와 명확한 진행 로그가 필요하다.
- clean WAV 생성이 선택 옵션이라 원본 README의 "음량 정리 기본 적용"과 Windows 실제 동작이 다르다.
- 원본의 `except Exception: print(...건너뜀)` 패턴은 Windows 구현 시 실패를 조용히 묻지 않도록 재설계해야 한다.
- `_upstream_compare_original`은 비교용 클론이며 배포 산출물에 포함하지 않는 것이 좋다.

## 테스트 방법

현재 테스트:

- `tests/test_paths.py`: Windows file URI의 공백/한글 인코딩
- `tests/test_ffmpeg_paths.py`: ffmpeg용 Windows native path
- `tests/test_fcp7.py`: XML pathurl/link/sample rate
- `tests/test_srt.py`: SRT time/grouping
- `tests/test_cut_analysis.py`: silence/repeated speech 후보
- `tests/test_timeline_mapper.py`: keep range와 timeline remap
- `tests/test_cli_parser.py`: CLI 옵션 파싱
- `tests/test_claude_assets.py`: `.claude` 자산 발견
- `tests/test_html_report.py`: HTML 리포트 요약 출력

필수 추가 테스트:

- 공백이 포함된 영상 경로
- 한글 파일명
- 5초 이하 짧은 영상
- 1시간 이상 긴 영상은 unit test 대신 fixture metadata/mock 또는 integration marker로 분리
- 음성이 없는 영상
- 배경음악이 큰 영상
- 여러 문장이 빠르게 이어지는 영상
- 반복 발화가 있는 영상
- false-start가 있는 영상
- 동일 폴더 일괄 처리
- Premiere Pro XML 불러오기 수동 smoke test
- XML의 미디어 경로 유효성
- SRT 타임코드와 컷 타임라인 일치
- Windows에서 FFmpeg 실행
- CUDA 사용 가능/불가 및 CPU fallback 동작

권장 명령:

```powershell
.\.venv\Scripts\python.exe -B -m pytest -q
.\.venv\Scripts\python.exe -B -m bibl_windows.cli doctor
.\.venv\Scripts\python.exe -B -m bibl_windows.cli probe ".\input\sample with space 한글.mp4"
.\.venv\Scripts\python.exe -B -m bibl_windows.cli run ".\input\sample with space 한글.mp4" --preset standard --limit-seconds 30 --allow-cpu-fallback --clean-wav
```

Premiere 수동 검증:

1. `output/<base>_cut.xml`을 Premiere Pro에서 `파일 > 가져오기`로 연다.
2. 미디어 offline 여부를 확인한다.
3. `output/<base>_cut.srt`를 가져와 컷 타임라인과 대략 맞는지 확인한다.
4. clean WAV 사용 시 XML의 오디오가 원본이 아니라 `*_cut_audio.wav`를 참조하는지 확인한다.

## 1단계 최소 구현 범위

대규모 포팅 전에 1단계에서 실제 수정할 최소 범위:

- CLI에 `--dry-run` 또는 `plan` 계열 분석 전용 모드를 추가해 STT/ffmpeg 변환 없이 출력 경로와 환경을 점검한다.
- `batch.ps1`이 개별 파일 실패를 기록하고 다음 파일로 진행하도록 한다.
- STT/CUDA 오류를 `RuntimeErrorWithHint` 계열로 감싸 어떤 단계에서 실패했는지 출력한다.
- HTML report에 keep/deletion 요약과 review 후보 수를 추가한다.
- `.claude`의 핵심 cut-editing 명령을 Windows CLI 기준으로 바로잡는다.
- README를 실제 Windows CLI/구현 상태 기준으로 정리한다.

이 범위는 원본 고급 기능을 한꺼번에 복사하지 않고, 현재 Windows 핵심 파이프라인을 깨지 않으면서 안정성을 높이는 데 집중한다.
