# 쇼츠 제작 (온디맨드)

**혼자 봐도 말이 되는 1분 쇼츠 5~10개**(9:16)를 만든다. 컷편집 파이프라인과 분리 — 사용자가 요청할 때만 실행.

## 제작 모드 (소스에 따라 선택)
- **모드 A · 컷 기반(권장)** — 그 영상에 Windows CLI 산출물(`_cut.xml`, `_cut_audio.wav`, `_transcript.json`, `_cut.srt`, `_workspace/<base>/10_research.md`)이 있으면 `bibl_windows.cli shorts`에 구간을 넘긴다. 구간은 기본적으로 **컷 타임라인 기준** = `_cut.srt` 시각으로 잡고, 필요하면 원본 시각과 구분해 적는다.
- **모드 B · 완성 롱폼** — 이미 편집 완료된 mp4만 있으면 완성본 기준 in/out을 잡아 `bibl_windows.cli shorts`로 9:16 XML을 만든다.

## 캡컷 납품 (요청 시)
캡컷은 XML을 못 연다 → 요청 시 `--render-mp4`로 9:16 MP4 렌더를 시도한다. NVENC/FFmpeg 환경에 좌우되므로 실패하면 XML/SRT 납품으로 전환하고 핸드오프에 기록한다.

## 입력
- **완성된 롱폼 영상**(사용자가 따로 업로드) + 그 전사본/자막. 원본 raw가 아니라 편집 끝난 버전 기준.
- 개수: 영상 맥락에 맞게 **5~10개** (콘텐츠가 풍부하면 많이, 적으면 적게).

## 비블 쇼츠 템플릿 (예시 측정값, 9:16)
| 요소 | 세로 위치 | 스타일 |
|------|----------|--------|
| 배경 | 전체 | 검정 |
| 제목 | 상단 11~24% | 흰색 굵게(Pretendard), 2줄, 가운데 — **수정 가능한 텍스트** |
| 영상 | 26~90% | 16:9를 좌우 ~50% 크롭(중앙)해 폭 꽉 채움(세로 줌인) |
| 자막 | ~61% | 흰색 굵게 6~7자 한 줄, 얇은 검정 외곽 |
| 워터마크 | ~94% | "비블 bibl" 회색, 가운데 — **수정 가능한 텍스트** |

**제목·워터마크·자막은 반드시 편집 가능한 텍스트**로 (영상에 굽지 말 것). 영상 레이아웃은 9:16 XML로, 텍스트는 비블 프리미어 템플릿/텍스트 레이어로 얹는다.

**프리미어 배치 실측값(short_01 확정, 1080×1920)**: 제목 그래픽 벡터모션 **Y=-460**(상단) / 워터마크 "비블 bibl" **Y=1341·비율 62**(맨 아래 작게, 비율 줄이면 Y도 같이 키움) / 캡션 하단 ~88%. **번인 자막 있는 소스**는 V1 모션 '아래쪽 자르기 ~12%'로 제거(소스마다 값 확인).

## 입력
- 원본 또는 완성 롱폼 영상, `output/<base>_transcript.md`(읽기 좋은 전사), `output/<base>_cut.srt`, `output/_workspace/<base>/10_research.md`(하이라이트·hook 점수), `output/_workspace/<base>/00_claude_context.md`.
- 전사/리서치 없으면 먼저 Windows `run` 파이프라인 또는 리서처를 실행한다.

## 절차

### 1. 5~10개 선정 (서로 다른 맥락)
완성 롱폼의 핵심 구간 중 **다른 각도**로 5~10개(콘텐츠 풍부하면 많이): 인트로훅 / 핵심논리 / 반전 / 구체사례 / 결론. 같은 주제 반복 금지.

### 2. 1분 윈도우로 확장 (맥락 완결)
각 하이라이트를 45~75초로 확장:
- **시작점** — 훅 문장이 클립 **첫 3초**에 오게. 그 앞에 꼭 필요한 한 문장만 설정으로. **자막(`_cut.srt`) 시작 시각에 스냅**해 문장 중간에서 시작하지 않기.
- **끝점** — 결론/임팩트 문장에서. 문장 중간에서 끊지 않기.
- 너무 길면(>75초) 곁가지를 빼서 압축, 너무 짧으면(<40초) 맥락을 더 포함.

### 3. 타임코드 기준 명시
완성 롱폼을 자르면 in-out은 완성본 기준이다. raw + 컷 산출물을 쓰면 원본 시각과 컷 타임라인 시각을 혼동하지 말고 반드시 표기한다.

### 4. 9:16 XML 생성 (mp4 아님 — 편집 가능)
```powershell
.\.venv\Scripts\python.exe -B -m bibl_windows.cli shorts "완성롱폼.mp4" "12:25-13:20" "23:10-24:05" --transcript ".\output\<base>_transcript.json"
```
→ `output/shorts/short_NN.xml` : 9:16 시퀀스 + 16:9 좌우크롭 세로꽉참. 프리미어로 불러와 수정.

### 5. 편집 가능 텍스트 (제목·자막·워터마크)
영상에 굽지 말고 텍스트로: **자막**=6~7자 한 줄 SRT(`short_NN.srt`, 프리미어 캡션), **제목·워터마크("비블 bibl")**=`50_shorts.md`에 텍스트로 적어 비블 프리미어 텍스트 템플릿(MOGRT)에 넣어 수정 가능하게. burn-in 금지.

### 6. 메모 → `output/_workspace/<base>/50_shorts.md`
```
## 쇼츠 1 — "유튜브는 조회수가 아니라 영업사원"
- in-out(원본): 03:35-04:30 (55초)
- 첫 3초 훅: "유튜브를 조회수로만 보면 망합니다"
- 선정 이유: 채널 핵심 철학, 제목과 직결
```

## 원칙 (왜)
- **첫 3초가 쇼츠의 생사** — 시작이 약하면 즉시 이탈. 가장 강한 단언/숫자/궁금증으로 연다.
- **자체 완결 > 1분 정확히** — 맥락이 끊기면 1분이어도 실패. 45~75초 안에서 완결점을 찾는다.
- **다양성** — 한 영상에서 다른 사람에게 꽂힐 5~10개의 다른 진입점.
- **편집 가능 텍스트** — 제목·워터마크·자막은 비블이 쇼츠마다 바꿔야 하므로 절대 굽지 않는다.
